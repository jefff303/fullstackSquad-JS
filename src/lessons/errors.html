La gestion d'erreurs en JavaScript

La gestion des erreurs est cruciale pour créer des applications robustes et fiables. Elle permet de contrôler le comportement de votre programme en cas d'imprévus ou d'erreurs d'exécution.

👉 Pourquoi gérer les erreurs ?

  • Empêcher le programme de crasher en cas de problème (ex: données manquantes, connexion échouée).
  • Fournir un retour clair et utile à l'utilisateur ou au développeur.
  • Assurer que le code s'exécute comme prévu, même dans des scénarios imprévus.

⚡ L'instruction `try...catch...finally`

Ce bloc de code permet de gérer les erreurs. Le code susceptible de provoquer une erreur est placé dans le bloc `try`. Si une `exception` est levée, le code du bloc `catch` s'exécute. Le bloc `finally` s'exécute toujours, qu'il y ait une erreur ou non. 

                
try {
  // Code à surveiller
  const resultat = 10 / uneVariableNonDefinie;
  console.log(resultat);
} catch (erreur) {
  // Code à exécuter si une erreur survient
  console.error('Une erreur est survenue :', erreur.message);
} finally {
  // Code qui s'exécute toujours
  console.log('Opération terminée.');
}
// Affiche :
// Une erreur est survenue : uneVariableNonDefinie is not defined
// Opération terminée.

🌱 Exceptions et le mot-clé `throw`

Une **exception** est un événement anormal qui se produit pendant l'exécution d'un programme. Le mot-clé `throw` vous permet de **lever** manuellement une exception, ce qui peut être utile pour valider des entrées. 

                
function verifierAge(age) {
  if (age < 18) {
    throw new Error('Vous devez avoir 18 ans ou plus.');
  }
  return 'Bienvenue !';
}

try {
  const message = verifierAge(15);
  console.log(message);
} catch (e) {
  console.error('Erreur :', e.message);
}
// Affiche : 'Erreur : Vous devez avoir 18 ans ou plus.'

✅ Bonnes pratiques

  • N'utilisez `try...catch` que pour des erreurs que vous ne pouvez pas éviter (erreurs d'exécution).
  • Utilisez `if` pour valider des conditions attendues (ex: une variable est-elle définie ?).
  • Levez des exceptions (`throw`) avec des messages clairs et descriptifs.
  • Utilisez le `finally` pour des opérations de nettoyage, comme la fermeture d'une connexion.
