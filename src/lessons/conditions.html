Les conditions : if/else et switch

Les conditions permettent à votre code de prendre des décisions. Elles exécutent différents blocs de code en fonction de l'évaluation d'une expression à `true` ou `false`.

👉 L'instruction if / else

La structure `if` est la plus courante. Elle exécute un bloc de code si une condition est vraie. On peut la combiner avec `else if` pour d'autres conditions, et `else` pour le cas par défaut. 

                
const temperature = 25;

if (temperature > 30) {
  console.log('Il fait très chaud !');
} else if (temperature >= 20) {
  console.log('Il fait bon.');
} else {
  console.log('Il fait frais.');
}
// Affiche 'Il fait bon.'

⚡ L'instruction switch

L'instruction `switch` est une alternative plus lisible aux chaînes de `if/else if` lorsque vous comparez une seule expression à plusieurs valeurs possibles. 

                
const jourDeLaSemaine = 'lundi';
let message;

switch (jourDeLaSemaine) {
  case 'lundi':
    message = 'Bonne semaine !';
    break;
  case 'vendredi':
    message = 'Le weekend approche !';
    break;
  case 'samedi':
  case 'dimanche':
    message = 'Profite bien du weekend !';
    break;
  default:
    message = 'Jour de la semaine classique.';
}
console.log(message); // Affiche 'Bonne semaine !'
  • `case` : Définit une valeur à comparer.
  • `break` : Stoppe l'exécution du `switch`. **Très important** pour éviter que le code ne s'exécute dans les cas suivants.
  • `default` : Exécute le code si aucun `case` ne correspond.

🌱 if/else vs. switch

  • `if/else` :
    • Idéal pour les **conditions complexes** basées sur des plages de valeurs (ex: `age > 18`).
    • Utile quand il y a peu de conditions.
  • `switch` :
    • Idéal pour les **conditions simples** qui testent une variable unique par rapport à plusieurs valeurs discrètes (ex: un statut, un mois de l'année).
    • Souvent plus lisible si vous avez de nombreux cas (`case`).

✅ Bonnes pratiques

  • Utilisez des noms de variables clairs pour vos conditions.
  • Indentez correctement votre code pour une meilleure lisibilité.
  • Assurez-vous de toujours inclure un `break` dans vos `case` d'un `switch` (sauf si vous souhaitez un comportement spécifique).
  • Le `default` dans un `switch` est une bonne pratique pour gérer les cas inattendus.
